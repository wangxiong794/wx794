内控易自动化测试

发表于 2019-06-28 |

[TOC]

序言

​ 自动化测试是测试工程师对系统测试时采用的一种方式，设计并编写自动化脚本完成一系列的测试操作，适用于不轻易修改或不做修改的功能和模块，若程序长期有调整，自动化测试脚本维护成本变高，增加项目的测试成本。常用应用分web应用，PC应用，APP方向，而根据不同的操作系统，自动化测试的工具不同。

​ 内控易项目分web应用和微信企业工作号两部分，微信企业工作号属于内嵌H5应用，因此整个应用可以划分为Web应用。对于Web应用，采用的自动化测试是引用selenium工具包调用浏览器的webdriver（浏览器的内核驱动）方法来满足测试需求。项目持续迭代更新，新增新功能的同时伴有优化改动，故自动化实现的逻辑功能较为简单，方便迭代更新后的修改，降低对自动化代码维护的工作量。

项目介绍

​ 项目为北京行控科技开发基于阿里云SaaS服务平台开发的行政事业单位内部控制Web应用-内控易，中国第一家专注于校园内控管理的SaaS服务提供商，致力于校园内控管理的持续创新，运用先进的互联网云应用技术，将国家政策与先进的管理思想融合，为学校、幼儿园及教育主管单位提供预算，收支，采购，合同，资产及项目一体化管理的内控管理云服务，帮助学校及主管单位建立智能能高校的内控管理云平台，防范风险的同时提升精细化管理能力。

​ 核心业务包括预算控制，收支管理，采购管理，合同管理，资产管理，项目管理，人事管理。为机关领导支持审批，包括Web桌面审批，微信审批，钉钉审批，从内控角度控制单位的业务类型，业务流程，关键控制文档，三重一大，风险评估，经济科目，组织架构，关键岗位，数字签名，扫脸认证授权等。

预算控制

​ 预算控制是企业根据预算规定的收入与支出标准检查和监督各个部门的生产经营活动的控制。其作用是保证各种活动或各个部门在充分达成既定目标，实现利润的过程中对经营资源的利用，使费用支出受到严格有效的约束。其预算内容包括收入预算、支出预算、现金预算、资金支出预算和生产负债预算等方面。

​ 预算控制分两类，一类已执行或执行中的预算称为预算管理，一类为未执行的预算，即未来的预算计划称为预算编制。

​ 预算管理的实际控制分项目和预算项，采用树状结构模式设计，预算项为项目树的末级，为预算项设立归口部门，财政账户，业务类型，业务额度，部门经济科目，功能科目，政府经济科目。系统所有的金钱绑定预算项，在预算项上进行增减，对预算项的调整是由机构/学校的审批流程来决定，可分已核销金额调整单据和可用预算调整单。

​ 预算编制时对来年的预算规划，分收入编制和支出编制两类，收入编制无需专门上报领导审批，由预算管理员可直接完成。支出编制需要单位长时间的讨论开会决定，分模板编制，一上填报，一上汇总，一下查询，二上填报，二上汇总，二下查询七大模块。预算管理员根据往年各部门的开支，整理来年的预算模板计划，即为模板编制。模板编制成功后，下发各部门，由各部门相关人员进行填报即为一上填报。填报后经大会商讨审批，由预算管理员将个预算的审批情况汇总，统计数据即为一上汇总。各个部门可在一上查询部门的预算审批情况，经过修改不合适的预算项再次填报预算项，即为二上填报。有机关领导相关人员组织二次会议对二上填报进行再审批，由预算管理员汇总即为二上汇总。各个部门的相关人员即可看见来年的部门实际预算情况，并在来年下发为实际预算项的使用与管理。

收支管理

​ 行政事业单位支出管理是依据现行财政经济制度规定、有关的行政事业开支标准、批准的行政事业单位预算和其他有关法律法规，对行政事业单位支出的项目、范围和标准等进行规范管理，将行政事业单位的财务行为限定在法律法规许可的范围内，督促各行政事业单位合理、节约使用资金，少花钱多办事，实现单位绩效最大化。行政事业单位收入管理是对行政事业性收费收入,属于政府非税收入的一种。

​ 收入管理通过收费标准单，收费单审批，收入登记单三类单据来控制学校的收入管理，其中收入登记单是最后财务入学校账户的最后一步，因此绑定预算项，只能由财务才能进行操作。

​ 支出管理通过事前经费单和报销单来控制，其中经费单可分会议费，培训费，差旅费，劳务费四大类，报销单除可对事前经费单发起报销外，可对合同及框架协议发起报销，也可直接对预算项发起无申请报销。另外由财务来核销与反核销，统一财务出入账。

采购管理

​ 采购管理是指对采购业务过程进行组织、实施与控制的管理过程。可分为自行采购和政府采购，需有事前经费申请，方能发起采购，由采购申请单和采购验收单共同控制采购管理，其中包括供应商管理，评审专家的管理。

合同管理

​ 合同管理是当事人双方或数方确定各自权利和义务关系的协议，虽不等于法律，但依法成立的合同具有法律约束力，工程合同属于经济合同的范畴，受经济和刑法法则的约束，合同管理主要是指项目管理人员根据合同进行工程项目的监督和言理，是法学、经济学理论和管理科学在组织实施合同中的具体运用。

​ 合同的发起依据与事前经费的申请，可分当前年度合同和跨年度合同，控制合同类型及印花税档案号，另外可根据用户的需求制定自己的合同版式。

资产管理

​ 资产管理业务是指资产管理人根据资产管理合同约定的方式、条件、要求及限制。资产可分固定资产和低值易耗品两类，数据来源可为事前经费单，采购审批单，采购验收单，也可自行生成相关数据。

​ 固定资产分六大模块，资产入库，资产建卡，资产领用，资产退还，资产处置，资产盘点。以及其他的基础数据，产品目录，计量单位，政采目录，资产分类，存放位置。以资产管理员为核心，对固定资产的统一管理。以入库单，领用单，退换单控制使用，采用专业设备对资产卡片的打印和盘点完成对固定资产的实际使用控制。

​ 低值易耗品分入库管理，领用出库，库存管理。相比固定资产，无需资产卡片。但需要期初库存，对库存的拆分，领用退还来控制低值易耗的使用。其数据来源也可来自事前经费单，采购审批单及验收单，亦可自行建立相关信息。

项目管理

​ 项目管理是满足或超越项目有关各方对项目的要求与期望，所开展的各种计划、组织、领导、控制等方面措施。分项目申报及中标单位通报两类单据进行项目控制。

自动化测试

​ 内控易自动化测试主要实现系统各类单据的自动生成及审批，包括预算调整单，收费标准单，收费审批单，收入登记单，事前经费单，报销申请单，采购审批单，采购验收单，资产入库单，资产建卡，资产领用单，资产退换单，资产处置单，项目申报单，中标单位通报单，框架协议单，事项申请单，议事会签单。其审批可同意，驳回，撤销，复制，作废，删除，抄送等操作，审批对接微信，可在微信审批，钉钉的审批还在后期研发中。

测试环境搭建

​ 自动化代码由python编写，采用eclipse的编译器编译，故此环境配置为python+selenium+browser，编译器推荐：eclipse.pydev+JDK

1、python安装，注意添加系统环境变量（PATH），点击详细安装教程。

2、browser选择，UI自动化的载体浏览器，建议选Chrome，Firefox，IE浏览器，其他浏览器需自己加载内核驱动。

以Chrome为例，下载安装Chrome后，下载webdriver并将该文件夹配置环境变量（PATH），点击下载链接，注意webdriver版本与浏览器版本一致，否则会运行失败。其他浏览器点击Firefox内核，IE内核。

3、selenium安装，selenium是python的一个工具包，用于模拟用户在浏览器的交互，命令行 pip install selenium即可。

4、JDK是eclipse的运行基础，点击安装教程。

5、eclipse下载，附教程无需安装，安装pydev模块，让eclipse支持编译python，点击安装教程。

​ 若只运行代码或使用其他编译器，前三步操作是不可或缺的，若有疑问，点击联系我。

代码解析

引用unittest框架，暂分两层，后期优化可划分三层结构。一层为底层代码，一层为用例代码。代码可见Github仅做参考，con_nky包为底层代码，即各个模块的操作事件集合，其中为解决函数不能反复调用问题，common_function文件为公共使用的函数，testcase是存放用例的集合包，用例文件格式固定，不能缺少setup和teardown部门，其中用例的命名必须要以test开头，否则不会执行。运行run_all文件可执行所有的用例，若同时需要用eclipse运行其他的代码，在命令行执行该文件即可。

三层架构的构思是公共函数包+功能事件包+测试用例包，是非常标准的pythonUI自动化框架，因公司使用此代码仅自己，故未再次推到重做。其中集成用例（run_all）执行文件中有两个函数，为集成执行用例和发送邮件函数。

系统迭代的更新范围较广较零散，故用例许多会执行不成功，需要进行调整。
